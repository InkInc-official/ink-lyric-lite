## これは何？

**Ink Lyric** は、AI音楽生成サービスへの歌詞提出を目的とした、**作詞に特化したWebアプリケーション**です。

「音数を確認しながら書く」と「本番歌詞を仕上げる」を画面上で分離することで、**作業用のメモを誤って提出してしまう事故をゼロにする**ことを目指して設計されました。

ブラウザだけで動作し、インストール不要・サーバー不要・完全無料です。

---

## 特徴

- **モーラ（音節）カウント** — 各行の音数をリアルタイムで表示
- **ドラフト／清書の分離** — 音数確認用の作業行と本番歌詞を物理的に別カラムで管理
- **ワンクリックコピー** — 清書カラムの内容を全文コピー。音楽生成サービスにそのまま貼り付けられる
- **セクションタグ挿入** — `[Verse]` `[Chorus]` などの構成タグをボタン一発で挿入
- **ノートメモ** — 各曲に制作メモを残せる（折りたたみ式）
- **カラム幅リサイズ** — 各カラムの境界をドラッグして自分好みのレイアウトに調整
- **自動保存** — 編集内容はブラウザに自動保存。ページを閉じても消えない
- **完全ローカル動作** — データはすべてブラウザ内に保存。外部サーバーへの送信なし

## 画面構成

```
┌──────────┬───────────────────────────┬──────────────────────┐
│  A       │  B                        │  C                   │
│          │                           │                      │
│ ノート    │ ドラフト                  │ 清書                 │
│ 一覧     │                           │                      │
│          │ ひらがな・カタカナで       │ 本番歌詞を入力        │
│          │ 音数を確認しながら書く     │                      │
│          │                           │ 右上のcopyボタンで   │
│          │ 左に音数（モーラ）表示     │ 全文コピー           │
│          │ 漢字の行はカウントしない   │                      │
└──────────┴───────────────────────────┴──────────────────────┘
```

### A — ノート一覧

曲ごとにノートを作成して管理します。付箋アイコンから制作メモを残すことができます。メモがあるノートには `※メモあり` と表示されます。

### B — ドラフトゾーン

ひらがな・カタカナで書きながら、各行の音数（モーラ数）をリアルタイムで確認するエリアです。漢字を含む行はカウントされないため、**「カウントが表示されている行＝音数確認済み」** という状態が一目でわかります。

セクションタグ（`[Verse]` `[Chorus]` など）はヘッダーのボタンから挿入できます。タグ行は `—` と表示され、カウント対象から外れます。

Bカラムの内容はリアルタイムでCカラムに反映されます。

### C — 清書ゾーン（finalゾーン）

音楽生成サービスに提出する本番歌詞を仕上げるエリアです。Bカラムの内容を引き継いだ上で、独立して編集できます。

右上の `copy` ボタンで全文をクリップボードにコピーできます。

## 使い方

### 基本的なワークフロー

```
1. Aカラムで新規ノートを作成

2. Bカラムでひらがな・カタカナを使い音数を確認しながら下書き

   例:
     [Verse]
     さくらなみきが    → 7音
     まんかいになって  → 7音
     みごろがすぎて    → 7音

3. 音数が固まったらCカラムで漢字・英語に変換して本番歌詞に仕上げる

   例:
     [Verse]
     桜並木が
     満開になって
     見頃が過ぎて

4. Cカラム右上の「copy」で全文コピー → 音楽生成サービスに貼り付けて提出
```

### セクションタグについて

多くのAI音楽生成サービスは、歌詞内のセクションタグを読み取って楽曲構成を判断します。Bカラムのヘッダーボタンからタグを挿入してください。

| タグ | 用途 |
|------|------|
| `[Main Theme]` | メインテーマ・全体のムード指定 |
| `[Intro]` | イントロ |
| `[Verse]` | Aメロ・Bメロ |
| `[Pre-Chorus]` | サビ前 |
| `[Chorus]` | サビ |
| `[Solo]` | 間奏・ソロ |
| `[Bridge]` | ブリッジ |
| `[Outro]` | アウトロ |

### モーラカウントについて

日本語の音数は「モーラ」という単位で数えます。

| 表記 | モーラ数 |
|------|----------|
| さくら | 3 |
| ありがとう | 5 |
| きぼうのあした | 7 |
| ゆっくり（拗音・促音含む） | 4 |

AI音楽生成サービスは音節の均一性を重視するため、各行のモーラ数を揃えることでメロディとの親和性が高まります。

## データの保存について

入力したすべてのデータは**ブラウザのlocalStorage**に保存されます。

- 外部サーバーへの送信は一切ありません
- ページを閉じても・PCを再起動しても保存されています
- **ブラウザのキャッシュ・サイトデータを削除するとすべてのデータが消えます**。大切な歌詞は定期的にCカラムの `copy` でテキストとして保存することを強くおすすめします
- 異なるブラウザ・デバイス間での同期はありません

### ⚠️ ブラウザごとの注意事項

| ブラウザ | 相性 | 備考 |
|---|---|---|
| Chrome | ◎ 推奨 | localStorageが安定して動作します |
| Firefox | ◎ 推奨 | localStorageが安定して動作します |
| Safari | △ 注意 | ITP（トラッキング防止機能）により、条件によってlocalStorageが**7日で自動削除**されることがあります |
| DuckDuckGo | ✕ 非推奨 | プライバシー保護のためlocalStorageが積極的に削除されます。データが予告なく消える可能性があります |
| プライベートブラウジング（シークレットモード） | ✕ 非推奨 | セッション終了時にlocalStorageが削除されます |

> **まとめ：Chrome または Firefox での使用を推奨します。**

## セットアップ（ローカルで動かす場合）

```bash
git clone <repository-url>
cd ink-lyric
npm install
npm run dev
```

`http://localhost:3000` にアクセス。

### ビルド（静的サイトとして出力）

```bash
npm run build
```

`out/` フォルダに静的ファイルが生成されます。Vercel・GitHub Pages・Netlify などにそのままデプロイできます。
