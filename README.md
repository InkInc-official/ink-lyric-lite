# Ink Lyric Lite

> **Ink Inc. presents** — AI Creation, Human Care. The Future Drawn Together.

作詞支援Webアプリケーション。音数カウントとドラフト／清書の分離ワークフローを提供する、**PC専用ブラウザアプリ**。

---

## 概要

Ink Lyric Lite は、SUNOなどのAI音楽生成サービスへの歌詞提出を念頭に置いた作詞支援ツールです。

「ドラフト（音数確認）」と「清書（本番歌詞）」を物理的に分離した3カラム構成により、**ひらがな作業行をSUNOに誤提出する事故をゼロにする**ことを主目的としています。

APIキー不要・サーバー不要。ブラウザとNode.jsだけで動作します。

## 機能一覧

### A カラム — ノート一覧

| 機能 | 説明 |
|------|------|
| ノート作成 | 右上の `+` ボタンで新規作成 |
| ノート選択 | クリックでB・Cカラムに展開 |
| タイトル編集 | トップバーの鉛筆アイコンから編集。Enterで確定、Escでキャンセル |
| ノート削除 | ホバー時に表示されるゴミ箱アイコンから削除（確認ダイアログあり） |
| ノートメモ | 付箋アイコンをクリックでメモパネルを展開。1ノートにつき1メモ保存可能 |
| ※メモあり表示 | メモが保存されているノートはタイトル下に `※メモあり` と表示 |
| 自動保存 | 編集から1.5秒後に自動保存（localStorage） |

---

### B カラム — ドラフトゾーン

**ひらがな・カタカナで音数を確認しながら作詞する作業エリア。**

| 機能 | 説明 |
|------|------|
| モーラカウント | 各行の左側にモーラ数（音数）をリアルタイム表示 |
| 漢字行はカウントしない | 漢字を含む行はカウント列が空欄（作業の混在を防止） |
| セクションタグ挿入 | ヘッダーのボタンから `[Verse]` `[Chorus]` 等を挿入。タグ行は `—` 表示 |
| Cカラムへ同期 | Bの内容はリアルタイムでCカラムに反映される |
| フッター | ノートタイトル・総文字数・総音数を表示 |

**対応セクションタグ：**
`[Main Theme]` / `[Intro]` / `[Verse]` / `[Pre-Chorus]` / `[Chorus]` / `[Solo]` / `[Bridge]` / `[Outro]`

---

### C カラム — 清書ゾーン（finalゾーン）

**SUNOなどのAI音楽生成サービスに提出する本番歌詞エリア。**

| 機能 | 説明 |
|------|------|
| B内容を初期反映 | Bカラムの内容がそのまま引き継がれ、独立して編集可能 |
| セクションタグボタンなし | 構成はBカラムで完結させるため非表示 |
| カウント列なし | シンプルなテキストエリアのみ |
| 全文コピー | ヘッダー右の `copy` ボタンでワンクリック全文コピー（提出用） |
| フッター | セクションタグを除いた歌詞行数を表示（`N lines`） |

---

### カラムリサイズ

各カラムの境界線をドラッグして幅を自由に調整できます。境界線はホバーでアンバー色にハイライトされます。

---

## データ保存仕様

すべてのデータは**ブラウザのlocalStorage**に保存されます。サーバーへの送信は一切ありません。

| データ | 保存キー |
|--------|----------|
| ノート本文 | `inklyric_notes` |
| ノートメモ | `inklyric_note_memos` |
| 最後に開いたノートID | `inklyric_active` |

### ノートファイルの内部フォーマット

ドラフトと清書は同一エントリにセパレータで区切って保存されます。

```
===DRAFT===
さくらなみきが
まんかいになって
===LYRIC===
[Verse]
桜並木が
満開になって
```

## 使い方（ワークフロー）

```
1. Aカラムで新規ノート作成
2. Bカラムでひらがな・カタカナを使い音数を確認しながら歌詞を下書き
   例:
     [Verse]
     さくらなみきが   ← 7音
     まんかいになって  ← 7音
3. 音数が確定したらCカラムで漢字・英語に変換して清書
   例:
     [Verse]
     桜並木が
     満開になって
4. Cカラム右上の「copy」で全文コピーしてSUNOに提出
```

## 設計思想

### ドラフト／清書の分離

音数確認用のひらがな行と本番歌詞が同一ファイルに混在すると、誤ってひらがな行をSUNOに提出してしまう事故が起きます。BとCを物理的に分離することで、提出ミスを構造的に防止します。

### 漢字行のカウント無効化

ひらがな・カタカナのみの行だけモーラカウントを表示します。「カウントがある行＝音数確認済み」という状態が視覚的に明確になります。

### API不要・ローカルデータ

外部サービスへの依存ゼロ。すべてのデータはブラウザのlocalStorageに保存されます。
